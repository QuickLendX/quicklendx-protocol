# QuickLendX Contract API Reference (rustdoc)

**Audience:** Downstream integrators — developers calling QuickLendX contracts from a Soroban client, another contract, or a backend service.

---

## Where to find the docs

The CI pipeline auto-publishes rustdoc for the **latest tagged release** to GitHub Pages every time a tag is pushed:

```
https://<org>.github.io/quicklendx/quicklendx_contracts/
```

Replace `<org>` with the GitHub organisation that owns this repository (e.g. `quicklendx-labs`). The URL is stable across releases — it always points to the most recently tagged version.

> **Tip:** Bookmark the top-level index at the URL above. Each tagged release overwrites the previous one, so you are always reading the current stable API.

---

## What is covered

The published docs are generated from the `quicklendx-contracts/` crate with `cargo doc`. They include:

| Section | What you will find |
|---|---|
| **Crate root** | Module overview and high-level design notes |
| **Public contract functions** | Every `#[contractimpl]` entry point with its parameters, return type, and doc-comment |
| **Error types** | The `Error` enum variants returned by each function |
| **Data types** | `InvoiceData`, `LoanTerms`, and every other `#[contracttype]` struct/enum |

Internal helpers and `pub(crate)` items are excluded from the published output.

---

## Reading a contract entry point

Each public function page follows this layout:

```
fn submit_invoice(env: Env, seller: Address, invoice_id: BytesN<32>, amount: i128) -> Result<(), Error>
```

- **`env: Env`** — injected by the Soroban host; you never pass this from a client.
- **`seller: Address`** — the Stellar account or contract submitting the invoice.
- **`invoice_id: BytesN<32>`** — a 32-byte unique identifier you generate off-chain.
- **`amount: i128`** — face value of the invoice in stroops (1 XLM = 10 000 000 stroops).
- **Returns** `Ok(())` on success, or one of the documented `Error` variants on failure.

The same pattern applies to every entry point. Check the **Error** section on each function page for the conditions that cause each variant.

## Calling a contract function from a Soroban client (Rust)

```rust
use soroban_sdk::{Address, BytesN, Env};

// `client` is generated by soroban-sdk's contractclient macro, or by
// running `soroban contract bindings rust` against the deployed contract.
let result = client.submit_invoice(
    &seller_address,          // Address
    &invoice_id_bytes,        // BytesN<32>
    &1_000_000_000_i128,      // 1 000 XLM face value
);

match result {
    Ok(()) => { /* invoice queued for financing */ }
    Err(e) => { /* inspect the Error variant */ }
}
```

> **No `std::`:** QuickLendX contracts are `#![no_std]`. All types shown above come from `soroban_sdk`; do not introduce `std` types in any code that touches the contract interface.

## Generating the docs locally

If you need to browse the docs offline, or you are reviewing a branch that has not been tagged yet:

```bash
cd quicklendx-contracts

# Generate HTML docs (output: target/doc/)
cargo doc --no-deps --target wasm32-unknown-unknown

# Open in your default browser (macOS / Linux)
open target/doc/quicklendx_contracts/index.html
```

Omit `--target wasm32-unknown-unknown` if you only need to read the docs and do not need to verify the WASM build at the same time.

---

## Staying up to date

The published URL is updated automatically on every tag push — no manual step is needed. To be notified of new releases, watch the repository on GitHub and select **Releases only**.

If the published URL returns a 404, the most likely cause is that no tag has been pushed yet for the current development cycle. Generate the docs locally using the command above, or check the repository's Actions tab to confirm the publish workflow ran successfully.
